## Vibe Coding技巧

1. 项目创建前
从项目创建前开始，就开始使用各平台AI进行项目架构，说明项目目标、功能、技术栈。让AI生成项目架构、项目目录以及项目代码。

整理上述所有问题和答案，形成一份文档。

创建一个项目目录，把文档丢进来。

2. 项目框架建设
从初始化项目开始，一点点的跟IDE的builder沟通。

比如：请基础Vite + Vue 3 + TypeScript + Element Plus初始化一个前端项目。检查代码，并打包成功。

在完成所有基础框架建设后，不要着急具体功能的开发，让AI先生成一个项目规则文档。里面包含项目目录结构、子项目技术栈、工作区、脚本命令、 开发要求、 质量与安全监测、文件命名规范、代码规范等等一系列要求。比如：项目A是公共核心库，任何改动完成都要build，才能验证功能。项目B是python，脚本需要先进venv虚拟环境才能执行。项目C是Vue，新增页面需要配置路由，请求需要使用封装好的axios等等。

保存到IDE的项目规则文档中。

3. 项目功能点开发
从项目功能开发开始，需要注意当前任务的上下文纯净，不要混合其他任务的上下文。

一次不要给它太多任务，一个任务会话以一个功能点为主，功能点完成之后，新建一个任务会话开始下一个功能点开发。

如果一个功能点设计前端、后端、数据库多个方面，需要从底部开始向上开发，就是先设计数据存储方式，在设计服务接口和逻辑，最后设计前端页面。保证整个任务会话的流程是通的，完成任务即可验证功能。

如果涉及多个子项目的联合改动，一定要注意协调，从一个项目开始改，不用担心影响到其他项目的功能跑不起来，一个项目一个项目关联的改过去。

比如：项目A是核心库，它被项目B、项目C引用，那么让AI先改项目A，告诉它不用着急验证其它项目是否跑不起来，先把项目A改完了，并完成打包验证。再在当前任务会话中，改项目B、项目C。它自己可以执行脚本并验证功能点。

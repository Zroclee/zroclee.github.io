# 如何开发全局命令行工具

要开发一个基于 Node.js 的全局命令行工具，包含多个子命令（例如启动后端服务、执行特定任务），你需要合理地组织项目结构、配置 package.json 的 bin 字段，并编写可执行脚本。下面我会一步步带你完成整个过程。

---

## 1. 初始化项目

创建项目文件夹，并在其中初始化 `package.json`：

```bash
mkdir my-cli-tool
cd my-cli-tool
npm init -y
```

修改 `package.json`，设置 `name`、`version`、`description` 等信息，并确保 `"type": "commonjs"` 或 `"module"`（根据你使用的模块系统，这里以 CommonJS 为例）。

---

## 2. 配置 bin 字段

`bin` 字段告诉 npm 在全局安装时，将指定的脚本链接到系统的 PATH 中。你可以有两种方式定义子命令：

- **方式 A：多个独立命令**  
  为每个子命令创建一个独立的可执行脚本，并在 `bin` 中分别映射。
- **方式 B：单个入口命令 + 子命令分发**  
  只创建一个入口脚本，然后通过第一个参数（如 `serve`、`task`）来区分执行不同逻辑。

推荐使用**方式 B**，更符合现代 CLI 工具的设计（如 `git`、`npm` 本身）。这样你的工具只有一个主命令，然后通过子命令扩展功能。

在 `package.json` 中这样配置：

```json
{
  "name": "my-cli",
  "version": "1.0.0",
  "bin": {
    "mycli": "./bin/mycli.js"
  },
  ...
}
```

这里我们定义了一个名为 `mycli` 的命令，它将指向 `./bin/mycli.js` 文件。

如果你希望有多个独立命令（例如 `mycli-server` 和 `mycli-task`），也可以这样：

```json
{
  "bin": {
    "mycli-server": "./bin/server.js",
    "mycli-task": "./bin/task.js"
  }
}
```

但为了体验更好，我们采用单个命令加子命令的方式。

---

## 3. 创建可执行脚本

在项目根目录创建 `bin` 文件夹，并在其中新建 `mycli.js` 文件。这个文件的开头必须包含 **shebang** 行，告诉系统用 Node.js 执行：

```bash
#!/usr/bin/env node
```

这是可执行脚本的标配。接下来，我们在该文件中编写逻辑，根据命令行参数调用不同的功能。

---

## 4. 实现命令逻辑

你可以手动解析 `process.argv`，也可以使用成熟的命令行解析库，例如：

- **[commander](https://www.npmjs.com/package/commander)** – 功能强大，API 简洁
- **[yargs](https://www.npmjs.com/package/yargs)** – 高度可定制
- **[cac](https://www.npmjs.com/package/cac)** – 轻量快速

这里以 **commander** 为例，演示如何定义 `serve` 和 `task` 两个子命令。

首先安装依赖：

```bash
npm install commander
```

然后编写 `bin/mycli.js`：

```javascript
#!/usr/bin/env node

const { program } = require('commander');

// 定义版本信息
program.version('1.0.0').description('我的 CLI 工具');

// 子命令：serve - 启动后端服务
program
  .command('serve [port]')
  .description('启动后端服务')
  .option('-h, --host <host>', '指定监听的主机', 'localhost')
  .action((port, options) => {
    const host = options.host;
    const serverPort = port || 3000;
    // 这里调用启动服务的函数，可以放在单独的模块中
    startServer(host, serverPort);
  });

// 子命令：task - 执行某个任务（例如数据迁移、清理等）
program
  .command('task <name>')
  .description('执行指定任务')
  .option('--force', '强制执行')
  .action((name, options) => {
    runTask(name, options.force);
  });

// 解析命令行参数
program.parse(process.argv);

// 如果没有输入任何子命令，显示帮助信息
if (!process.argv.slice(2).length) {
  program.outputHelp();
}

// ----- 实际功能实现（可以放在其他模块中）-----
function startServer(host, port) {
  console.log(`启动服务在 ${host}:${port}`);
  // 例如：引入你的后端服务模块并启动
  // require('../server').listen(port, host);
}

function runTask(taskName, force) {
  console.log(`执行任务: ${taskName}, 强制模式: ${force}`);
  // 根据 taskName 调用不同的任务函数
  // 可以做成配置映射，或动态 require 任务模块
}
```

如果你的功能代码较多，建议将 `startServer` 和 `runTask` 等具体实现放到 `lib/` 目录下的独立文件中，然后在 `bin/mycli.js` 中引入，保持入口文件的简洁。

---

## 5. 本地测试

在发布到 npm 之前，你可以使用 `npm link` 在本地全局安装这个正在开发的包，进行测试。

在项目根目录运行：

```bash
npm link
# 或
sudo npm link
```

这会在全局 `node_modules` 中创建一个符号链接，并且 `mycli` 命令就可用。你可以直接在终端中尝试：

```bash
mycli serve 8080 --host 0.0.0.0
mycli task backup --force
mycli --help
```

如果需要解除链接，运行 `npm unlink -g my-cli-tool`（注意包名）。

---

## 6. 发布与全局安装

当你完成开发并测试通过后，可以发布到 npm 仓库（需要先注册 npm 账号）：

```bash
npm publish
```

之后用户就可以通过以下命令全局安装并使用：

```bash
npm install -g my-cli-tool
```

安装后，用户就能直接运行 `mycli` 命令了。

---

## 补充：多命令独立脚本的写法

如果你选择为每个子命令创建独立的脚本（方式 A），可以这样组织：

- `bin/serve.js`
- `bin/task.js`

每个文件同样以 `#!/usr/bin/env node` 开头，并在 `package.json` 中配置：

```json
"bin": {
  "mycli-serve": "./bin/serve.js",
  "mycli-task": "./bin/task.js"
}
```

然后分别实现各自逻辑。这种方式比较简单直接，但用户需要记住多个命令名称（例如 `mycli-serve`、`mycli-task`），不如子命令方式优雅。

## 项目结构示例

```
my-cli-tool/
├── bin/
│   └── mycli.js          # 主入口脚本
├── lib/
│   ├── server.js         # 后端服务启动模块
│   └── tasks.js          # 任务函数集合
├── package.json
└── README.md
```

## 注意事项

1. **权限**：确保 `bin/mycli.js` 文件具有可执行权限（Unix 下可运行 `chmod +x bin/mycli.js`）。
2. **跨平台**：`#!/usr/bin/env node` 可以在 Unix 和 Windows 上正常工作。
3. **错误处理**：在脚本中添加适当的错误捕获和友好提示。
4. **异步逻辑**：如果子命令中有异步操作（如启动 HTTP 服务器），确保进程不会立即退出，或者使用 `process.on` 处理。
5. **帮助信息**：使用 commander 或 yargs 自动生成的帮助信息可以让你的工具更易用。
